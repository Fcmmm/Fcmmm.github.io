当 Claude Code 用两小时完成我两周的工作 — 一个 Vibe Coding 小白的震撼与思考

发表于2026-05-28|更新于2026-05-28|生活记录
|总字数:871|阅读时长:2分钟|浏览量:

先说结论：这个博客，是 AI 写的

你现在看到的这个网站——从 Hexo 框架搭建、Butterfly 主题配置、8 篇文章的生成、评论系统（Giscus）接入、到 GitHub Pages + 阿里云自定义域名部署——全部由 Claude Code + DeepSeek V4 Pro 在约两小时内完成。

而我的角色，只是一个”翻译官”：用自然语言描述需求，然后看着 AI 一步步把事情做完。

一年前的我：两周的噩梦

大约一年前，我尝试过搭建一个类似的个人博客。

当时的流程是这样的：

  • 搜教程 → 版本不对 → 继续搜 → 配置文件报错 → 百度/Google → 改配置 → 又报别的错
  • 主题安装 → 依赖冲突 → 翻 GitHub Issues → 凌晨两点还在找 bug
  • 想加个评论功能 → 各种第三方服务注册 → 密钥配错 → 调试到崩溃

断断续续折腾了两周，最后还是有各种小 bug 不断。那种挫败感，做过前端的人应该都懂。

这一次：两个小时的从容

我打开终端，对 Claude Code 说：”仿照这个博客网站，帮我在 Blog02 文件夹里做一个功能结构相仿的网站。”

然后：

  1. 环境搭建 — AI 自动检测环境、安装 Hexo、克隆 Butterfly 主题，一气呵成
  2. 配置生成 — 数百行的 _config.yml，作者信息、菜单结构、第三方插件……全部自动填充
  3. 文章生成 — 8 篇示例文章，覆盖技术文档、前端开发、DevOps 等多个分类，带 YAML frontmatter、标签、封面图
  4. 评论系统 — 从 Twikoo 到 Giscus，AI 帮我评估方案、完成接入、排查部署问题
  5. 部署上线 — GitHub Pages + 阿里云域名配置，甚至连 CNAME 编码问题都被 AI 排查出来了

整个过程，我做的事情就是：

  • 描述需求
  • 复制粘贴（偶尔）
  • 说”这个不对，改一下”
  • 喝咖啡，看着 AI 工作

技术的进步，真的令人发怵

这不是什么高深的算法或学术突破。这就是一个普通开发者 + 一个 AI 编程助手，在 2026 年的日常。

Claude Code 不是简单地把代码补全——它能理解上下文、规划架构、排查错误、迭代修复。DeepSeek V4 Pro 作为底层模型，提供了强大的推理和代码生成能力。两者结合，产生了一种全新的编程范式。

有人说这叫 “Vibe Coding”——你不需要精确地知道每一行代码怎么写，你只需要清楚地表达你想要什么，AI 来负责实现。

听起来很玄乎？但这就是我现在做的事情。

未来：只会更快、更强、更多元

这还只是 2026 年。我还是一个第一次尝试 Vibe Coding 的新手。

想想看——当这种技术再发展一年、两年、五年：

  • 编程的门槛会降到多低？
  • 一个人 + AI 的生产力上限会提到多高？
  • “会写代码”这件事的定义，会不会彻底改变？

技术的进步速度让人发怵，但也让人兴奋。

最后

我写下这些，不是要贩卖焦虑。

我只是想记录下这一刻——一个写了两周 bug 还调不好博客的人，在一年后，用 AI 两小时搞定了同样的事情。

如果你也在用 AI 工具辅助编程，欢迎在评论区聊聊你的体验。
