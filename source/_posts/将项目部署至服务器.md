## 今天把项目部署在了阿里云服务器上[阿里云服务器300元优惠券](http://i8c.cn/zuCii)，大体过程：

 **部署过程**：

```text
主项目：Hexo + Butterfly
访问： https://fcmmm.xyz/

```

最终部署结构：

```text
/www/wwwroot/fcmmm.xyz/
├── main/        # Hexo 主站 public/ 目录里的内容
```

Hexo 的静态构建产物默认生成在 `public/`，部署时应同步 `public/` 里的内容，而不是整个 Hexo 源码目录。([Hexo](https://hexo.io/docs/generating?utm_source=chatgpt.com)) GitHub Actions 里使用 `actions/checkout` 是为了把仓库代码检出到工作区，后续步骤才能构建或同步文件。([GitHub](https://github.com/actions/checkout?utm_source=chatgpt.com)) GitHub Secrets 用于保存 SSH 私钥、服务器 IP 等敏感信息。([GitHub Docs](https://docs.github.com/actions/security-guides/using-secrets-in-github-actions?utm_source=chatgpt.com))

# 一、服务器目录准备

先用 `root` 登录服务器：

```bat
ssh root@114.215.125.28
```

进入服务器后执行：

```bash
mkdir -p /www/wwwroot/fcmmm.xyz/main
```

确认目录：

```bash
ls -la /www/wwwroot/fcmmm.xyz/
```

应该看到：

```text
main/
```

------

# 二、创建 deploy 部署用户

仍然在服务器里执行：

```bash
id deploy || useradd -m -s /bin/bash deploy
```

创建 SSH 目录：

```bash
mkdir -p /home/deploy/.ssh
touch /home/deploy/.ssh/authorized_keys
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys
```

给网站目录权限：

```bash
chown -R deploy:deploy /www/wwwroot/fcmmm.xyz
chmod -R 755 /www/wwwroot/fcmmm.xyz
```

退出服务器：

```bash
exit
```

# 三、Windows 本地生成 SSH 密钥

在你的 Windows CMD 中执行：

```bat
ssh-keygen -t ed25519 -C "github-actions-deploy-fcmmm" -f github-actions-deploy-fcmmm
```

生成两个文件：

```text
github-actions-deploy-fcmmm       # 私钥，放 GitHub Secrets
github-actions-deploy-fcmmm.pub   # 公钥，放服务器
```

# 四、把公钥写入服务器

你是 Windows CMD，不能用 `cat`，要用 `type`。

执行：

```bat
type github-actions-deploy-fcmmm.pub | ssh root@114.215.125.28 "cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

然后测试：

```bat
ssh -i github-actions-deploy-fcmmm deploy@114.215.125.28
```

如果成功，会进入服务器，例如：

```text
[deploy@xxx ~]$
```

如果还是：

```text
Permission denied (publickey,gssapi-keyex,gssapi-with-mic)
```

说明密钥仍然没配置成功，需要检查：

```bash
cat /home/deploy/.ssh/authorized_keys
ls -la /home/deploy/.ssh
```

成功登录后退出：

```bash
exit
```

# 五、宝塔 Nginx 配置

进入宝塔面板：

```text
网站 → fcmmm.xyz → 设置 → 配置文件
```

使用下面这份配置。它适配：

```text
/www/wwwroot/fcmmm.xyz/main      → 主站
server
{
    listen 80;
    listen 443 ssl;
    listen 443 quic;
    http2 on;
    http3 on;

    server_name fcmmm.xyz www.fcmmm.xyz;

    # 主站根目录：Hexo + Butterfly 构建后的 public/ 内容
    root /www/wwwroot/fcmmm.xyz/main;
    index index.html index.htm default.htm default.html;

    # 宝塔扩展配置
    include /www/server/panel/vhost/nginx/extension/fcmmm.xyz/*.conf;

    # CERT-APPLY-CHECK--START
    include /www/server/panel/vhost/nginx/well-known/fcmmm.xyz.conf;
    # CERT-APPLY-CHECK--END

    # SSL-START
    # error_page 404/404.html;
    ssl_certificate        /www/server/panel/vhost/cert/fcmmm.xyz/fullchain.pem;
    ssl_certificate_key    /www/server/panel/vhost/cert/fcmmm.xyz/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers EECDH+CHACHA20:EECDH+CHACHA20-draft:EECDH+AES128:RSA+AES128:EECDH+AES256:RSA+AES256:EECDH+3DES:RSA+3DES:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_tickets on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    add_header Strict-Transport-Security "max-age=31536000";
    add_header Alt-Svc 'quic=":443"; h3=":443"; h3-29=":443"; h3-27=":443"; h3-25=":443"; h3-T050=":443"; h3-Q050=":443"; h3-Q049=":443"; h3-Q048=":443"; h3-Q046=":443"; h3-Q043=":443"';
    quic_retry on;
    quic_gso on;
    ssl_early_data on;
    error_page 497 https://$host$request_uri;
    # SSL-END

    # ERROR-PAGE-START
    # error_page 404 /404.html;
    # error_page 502 /502.html;
    # ERROR-PAGE-END

    # REWRITE-START
    include /www/server/panel/vhost/rewrite/html_fcmmm.xyz.conf;
    # REWRITE-END


    # Hexo 主站
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|webp|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public";
        try_files $uri =404;
    }

    # 禁止访问敏感文件
    location ~* (\.user.ini|\.htaccess|\.htpasswd|\.env.*|\.project|\.bashrc|\.bash_profile|\.bash_logout|\.DS_Store|\.gitignore|\.gitattributes|LICENSE|README\.md|CLAUDE\.md|CHANGELOG\.md|CHANGELOG|CONTRIBUTING\.md|TODO\.md|FAQ\.md|composer\.json|composer\.lock|package(-lock)?\.json|yarn\.lock|pnpm-lock\.yaml|\.\w+~|\.swp|\.swo|\.bak(up)?|\.old|\.tmp|\.temp|\.log|\.sql(\.gz)?|docker-compose\.yml|docker\.env|Dockerfile|\.csproj|\.sln|Cargo\.toml|Cargo\.lock|go\.mod|go\.sum|phpunit\.xml|pom\.xml|build\.gradl|pyproject\.toml|requirements\.txt|application(-\w+)?\.(ya?ml|properties))$
    {
        return 404;
    }

    # 禁止访问敏感目录
    location ~* /(\.git|\.svn|\.bzr|\.vscode|\.claude|\.idea|\.ssh|\.github|\.npm|\.yarn|\.pnpm|\.cache|\.husky|\.turbo|\.next|\.nuxt|node_modules|runtime)/ {
        return 404;
    }

    # SSL 验证目录
    location ~ \.well-known {
        allow all;
    }

    if ( $uri ~ "^/\.well-known/.*\.(php|jsp|py|js|css|lua|ts|go|zip|tar\.gz|rar|7z|sql|bak)$" ) {
        return 403;
    }

    access_log  /www/wwwlogs/fcmmm.xyz.log;
    error_log   /www/wwwlogs/fcmmm.xyz.error.log;
}
```

保存后 SSH 执行：

```bash
nginx -t
systemctl reload nginx
```

# 七、GitHub Secrets 配置

仓库添加 Secrets：

```text
Fcmmm.github.io
```

进入仓库：

```text
Settings → Secrets and variables → Actions → New repository secret
```

添加：

```text
ALIYUN_HOST       114.215.125.28
ALIYUN_USER       deploy
ALIYUN_PORT       22
ALIYUN_SSH_KEY    github-actions-deploy-fcmmm 私钥完整内容
```

注意：

`ALIYUN_SSH_KEY` 填的是私钥，不是 `.pub` 公钥。

在 Windows CMD 查看私钥内容：

```bat
type github-actions-deploy-fcmmm
```

复制完整内容，包含：

```text
-----BEGIN OPENSSH PRIVATE KEY-----
...
-----END OPENSSH PRIVATE KEY-----
```

# 八、主项目 Fcmmm.github.io 的 GitHub Actions

在 `Fcmmm.github.io` 仓库中新建：

```text
.github/workflows/deploy.yml
```

内容：

```yaml
name: Deploy Hexo Main Site to Aliyun

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout source
        uses: actions/checkout@v4
        with:
          submodules: recursive

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm install

      - name: Build Hexo
        run: |
          npx hexo clean
          npx hexo generate

      - name: Configure SSH
        run: |
          mkdir -p ~/.ssh
          echo "${{ secrets.ALIYUN_SSH_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519
          ssh-keyscan -H -p "${{ secrets.ALIYUN_PORT }}" "${{ secrets.ALIYUN_HOST }}" >> ~/.ssh/known_hosts

      - name: Deploy to Aliyun
        run: |
          rsync -az --delete \
            -e "ssh -p ${{ secrets.ALIYUN_PORT }}" \
            public/ \
            ${{ secrets.ALIYUN_USER }}@${{ secrets.ALIYUN_HOST }}:/www/wwwroot/fcmmm.xyz/main/
```

如果你的主分支是 `master`，把：

```yaml
branches: [main]
```

改成：

```yaml
branches: [master]
```

# 十、完整操作顺序

按这个顺序做：

```text

8. 本地确认 ssh -i github-actions-deploy-fcmmm deploy@114.215.125.28 能登录

10. 两个 GitHub 仓库分别添加 Secrets
11. Fcmmm.github.io 添加 Hexo 部署 workflow
12. push 仓库
13. 检查 GitHub Actions 是否成功
14. 访问 fcmmm.xyz
```
